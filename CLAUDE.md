## About

A Next.js 14 app for fast repetition of vocabulary from Lingualeo (a language learning service). Words are fetched from the Lingualeo API and stored locally in `auto-generated/words.json`. The UI lets users cycle through flashcard-style word cards.

## Package Manager

Use **pnpm** (not npm or yarn). Node >=24.12.0, pnpm >=10.26.2 required.

## Common Commands

```bash
pnpm dev              # Start dev server at localhost:3000
pnpm build            # Production build
pnpm lint             # ESLint
pnpm tsc              # TypeScript type check (no emit)
pnpm test             # Run Vitest in watch mode
pnpm test:ci          # Run Vitest once (CI)
pnpm test:update      # Update Vitest snapshots
pnpm e2e              # Playwright tests with UI
pnpm e2e:ci           # Playwright tests headless
pnpm prettier-format  # Format all files

# Word generation (requires .env with ACCOUNT_EMAIL and ACCOUNT_PASSWORD)
pnpm generate-words   # Fetch words from Lingualeo API → auto-generated/words.json
pnpm sync             # generate-words + generate extra images

# Content parsing (produces ./content/FOUND.json)
pnpm parse-youtube <id>
pnpm parse-text <path>
pnpm parse-pdf <path>
pnpm parse-url <url>
```

## URL Routes

- `/` — main word card view (all words)
- `/last/20` — last 20 words by `added` timestamp
- `/last/1w` — words added in last 1 week (time period syntax)
- `/offsetlimit/0/2000` — words by offset and limit
- `/all` — all words list view
- `/irregular` — irregular verbs practice
- `/verbs-to-ing` — gerund/ing form practice
- `/useful` — useful content page
- `/debug` — debug view

## Architecture

### Data Flow

1. `auto-generated/words.json` — source of truth for word data (committed, auto-generated by `scripts/generate-words.ts`)
2. `App` component (`components/complex/App/App.tsx`) — loads words from JSON, applies slicing based on route params, dispatches to Redux store
3. Redux store persists state via `redux-persist` to `localStorage`
4. `StoreProvider` (`app/StoreProvider.tsx`) — handles version-based cache busting: on version mismatch or URL change, flushes persist and reloads

### Redux Store (`store/`)

Three slices:

- `wordsSlice` — core flashcard state: `allWords`, `currentWord`, `finishedWords`, `wrongWords`, `isCheckState`, `isByWrongWords`. Handles card navigation logic (ok/no/back), cycling through wrong words after completing all words
- `settingsSlice` — language direction (EN↔RU), speed mode, sound settings
- `irregularWordsSlice` — separate state for irregular verbs practice

State is persisted to localStorage via `redux-persist`.

### Component Structure

- `components/simple/` — pure presentational components (buttons, loading)
- `components/complex/` — stateful components connected to Redux (App, Menu, WordView, TopMenu, BottomControl, SoundControl, etc.)

### Models (`models/`)

Zod schemas for runtime validation:

- `Word` — id, wordValue, combinedTranslation, transcription, picture (PNG URL), pronunciation (MP3 URL), created, added
- `Group` — wrapper grouping words from the API
- `IrregularWord` — verb forms for irregular verbs

### Pre-commit Hook

The pre-commit hook (`.husky/pre-commit`) runs automatically:

1. Bumps patch version in `package.json`
2. Runs `prettier-format`
3. Stages all changes
4. Runs `tsc`

### CI (GitHub Actions)

On PRs: type-check (`tsc`), lint, unit tests (`test:ci`), and Playwright e2e tests run in parallel jobs.

## Key Conventions

- Path alias `@/` maps to the project root (configured via `tsconfig.json`)
- Prettier config: single quotes, no semicolons, 2-space indent, trailing commas (ES5)
- Tests use Vitest + `@testing-library/react` + jsdom; snapshot tests are common for components
- `tsx` is used to run TypeScript scripts directly (no compilation step needed)
